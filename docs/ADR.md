## Overview

This file collects all Architecture Decision Records for the deckhouse-helm-generator project. Each record documents a significant technical decision: the context that required it, what was decided, and its current status.

Statuses: **Accepted** (in effect), **Proposed** (drafted, not yet implemented).

| ADR-005 | 2026-02-27 | Phase 2 generators split into 3 tiers (infrastructure, detection, advanced) | Accepted | Manages delivery risk; each tier ships as a separate PR batch |

## Phase 4: Source Expansion (ADR-014 – ADR-016)

| ADR | Date | Decision | Status | Context |
|-----|------|----------|--------|---------|
| ADR-014 | 2026-03-28 | Use `k8s.io/client-go` dynamic client for cluster extraction | Proposed | `k8s.io/apimachinery` already present; `client-go` shares transitive deps. Built-in auth (kubeconfig, OIDC, exec, in-cluster), pagination, and CRD discovery justify ~15–20 MB binary increase |
| ADR-015 | 2026-03-28 | Use `go-git/v5` for git operations; shell-exec `kustomize` binary for kustomize build | Proposed | Pure Go clone works in scratch Docker images; `kustomize build` requires the external binary regardless |
| ADR-016 | 2026-03-28 | Multi-source merge as separate `merger.go` component; default strategy `file-wins`; no three-way merge in v1 | Proposed | Deduplication by `ResourceKey`; conflicts produce `MergeConflict` structs. Three-way merge deferred — high complexity, low ROI for v1 |

## Phase 5: Advanced Analysis (ADR-017 – ADR-024)

| ADR | Date | Decision | Status | Context |
|-----|------|----------|--------|---------|
| ADR-017 | 2026-03-28 | `dhg fix` as a top-level Cobra command (not `generate --fix`) | Proposed | `fix` mutates input manifests; `generate` produces a Helm chart — different I/O contracts warrant separate commands |
| ADR-018 | 2026-03-28 | `GenericCRDProcessor` as a dynamic fallback for unknown CRDs | Proposed | Walks `spec` fields at runtime; generates `{{ toYaml .Values.<kind>.spec }}` template. Typed processors remain for known kinds (ADR-003) |
| ADR-019 | 2026-03-28 | Native DOT text generation — no graphviz library dependency | Proposed | DOT is a trivial text format; users render with `dot -Tpng`. Avoids CGo dependency and keeps binary portable |
| ADR-020 | 2026-03-28 | One file per secret provider in `pkg/generator/secrets/`; common `SecretStrategy` interface | Proposed | ESO, Sealed Secrets, Vault CSI, Vault Agent, SOPS, Reloader each isolated. Selection via `--secret-strategy` flag |
| ADR-021 | 2026-03-28 | Service mesh generators (Istio, Linkerd) as pipeline post-processors (copy-on-write) | Proposed | Follows Phase 2/2.5 pattern — add templates and values to `GeneratedChart`; no new pipeline stage |
| ADR-022 | 2026-03-28 | Plugin system via subprocess JSON (stdin/stdout); not Go `plugin.Open` or WASM | Proposed | Go plugins require exact Go version match and are Linux/macOS-only. Subprocess pattern is proven (Terraform, Helm, kubectl plugins) |
| ADR-023 | 2026-03-28 | Worker pool per pipeline stage; parallel in Process and Generate stages; sequential Analyze | Proposed | Per-resource parallelism in Stage 2 (processors are independent). Graph construction in Stage 3 must be sequential. Bounded goroutine pool (`GOMAXPROCS`) |
| ADR-024 | 2026-03-28 | GoDoc on `pkg.go.dev` as primary API documentation; no separate docs site for v1.0.0 | Proposed | Auto-published from public Go module; low maintenance overhead. Revisit if plugin ecosystem grows |

## Phase 5.5–5.9: Smart Analysis, Templating, Secrets, Mesh, Observability (ADR-039 – ADR-044)

| ADR | Date | Decision | Status | Context |
|-----|------|----------|--------|---------|
| ADR-039 | 2026-03-29 | JSON Schema validation depth limited to 5 levels of nesting | Accepted | Prevents combinatorial explosion in schema generation; covers all practical Helm values structures |
| ADR-040 | 2026-03-29 | `PostRendererEnv` typed struct instead of embedding `KustomizeDir` in post-renderer generator | Accepted | Avoids struct aliasing across generator calls; aligns with copy-on-write contract (ADR-008) |
| ADR-041 | 2026-03-29 | Operator scaffold uses pure YAML templates, not Kubebuilder markers | Accepted | Kubebuilder markers require the `controller-gen` binary; pure YAML works anywhere, easier to test |
| ADR-042 | 2026-03-29 | Secret strategies (ESO, Sealed Secrets, Vault CSI, SOPS) are mutually exclusive via `--secret-strategy` flag | Accepted | Multiple strategies in one chart produce conflicting Secret references; single-strategy selection avoids ambiguity |
| ADR-043 | 2026-03-29 | Istio and Linkerd generators are independent modules, not a shared abstraction | Accepted | The two meshes differ enough in CRDs (VirtualService vs SMI) that a forced shared interface would be leaky |
| ADR-044 | 2026-03-29 | OpenTelemetry language detection uses image name heuristics, not runtime analysis | Accepted | Runtime analysis requires cluster access; image name patterns (`*-java`, `*-python`, etc.) work in file-only mode |

## Phase 10: Database Operators (ADR-032 – ADR-034)

| ADR | Date | Decision | Status | Context |
|-----|------|----------|--------|---------|
| ADR-032 | 2026-03-28 | Database processors in subdirectories by operator family (`db/postgres/`, `db/mysql/`, `db/redis/`, etc.) | Proposed | 51 tasks across 12+ operators; flat `k8s/` directory is unscalable at this size |
